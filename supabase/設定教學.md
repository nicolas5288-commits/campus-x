# Campus X — Supabase 設定教學（約 5 分鐘）

做完這 4 步，網站就從「本機展示模式」切換到「雲端資料庫」，學生就能真的註冊登入、收藏會跟著帳號走。

## 步驟 1：建立 Supabase 專案
1. 到 https://supabase.com → 登入（你購物趣那個帳號就行）
2. 右上 **New project** → 專案名打 `campus-x` → 選一個資料庫密碼（自己記著，前端不會用到）→ Region 選 `Northeast Asia (Tokyo)`（離台灣近）
3. 按 Create，等 1~2 分鐘跑好

## 步驟 2：建立資料表（貼 SQL）
1. 左側選單 **SQL Editor** → **New query**
2. 打開 `supabase/schema.sql`，**先把裡面兩個地方的 email 改成你的**：
   - `is_admin()` 函式裡的 `CHANGE_ME@example.com`
   - （schema 只有這一處 email，改一次即可）
3. 整段複製貼上 → 按 **Run**。看到 Success 就成功了。

## 步驟 3：拿 URL 和 Key
1. 左側 **Project Settings（齒輪）→ API**
2. 複製兩個值：
   - **Project URL**（像 `https://xxxx.supabase.co`）
   - **anon / publishable key**（很長一串，這個設計來公開，安全靠剛剛 SQL 建的 RLS 規則）

## 步驟 4：填進 config.js
打開 `config.js`，把三個值換掉：
```js
window.CAMPUSX_CONFIG = {
  SUPABASE_URL: "貼上你的 Project URL",
  SUPABASE_KEY: "貼上你的 anon key",
  ADMIN_EMAIL: "你的 email（要跟 SQL 裡改的那個一致）",
};
```
存檔，重新整理網站 → 左下角會從「🖥️ 本機展示模式」變成「☁️ 雲端資料庫」，就成功了。

## 常見狀況
- **註冊後說要收驗證信**：Supabase 預設會寄 email 驗證。若你想學生註冊完直接能用，去 **Authentication → Providers → Email** 把「Confirm email」關掉（測試期方便；正式上線建議開著防假帳號）。
- **登入後收藏沒存**：確認 SQL 有跑成功（favorites 表存在），且你有登入。
- **審核權限**：只有 `ADMIN_EMAIL` 那個帳號登入後，才看得到 pending 資料、才能改狀態（Phase 3 後台會用到）。

## 資料表一覽（這次建的）
| 表 | 用途 | 誰能寫 |
|---|---|---|
| programs | 計畫本體 | 廠商投稿（免登入，只能建 pending）；管理員審核改狀態 |
| favorites | 學生收藏 | 學生本人 |
| reviews | 學長姐心得（Phase 3 UI） | 登入學生（只能建 pending）；管理員審核 |
| subscriptions | Email 訂閱名單 | 任何人可訂；只有管理員看得到名單 |
